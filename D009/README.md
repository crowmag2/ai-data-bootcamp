## 3. 처리 결정과 근거 (5줄로)
| 단계 | 액션 | 근거 | 결과 |
|---|---|---|---|
| 1. 중복 제거 | 완전 중복 행 drop | 동일 키 동일 값은 시스템 입력 오류로 간주 | 321건 제거됨 |
| 2. 문자열 정제 | item '케익'을 '케이크'로. store_id " S01 " 공백 제거 | 영어 표기 문제. 공백 문제 | '케이크'로 통일됨. 공백 제거됨. |
| 3. 날짜 파싱 | date_str를 datetime으로 | 집계와 분석 시 datetime 필요 | 날짜형 데이터로 변환됨 |
| 4. 이상치 처리 | revenue 음수 분리 후 분석에서 제외 | 분석 목적으로 환불 또는 실수는 별도 보관 | 1건 보관 |
| 5. 결측치 처리 | revenue 결측 행 제거 | 결측은 결제 실패 로그로 추정(MCAR 가정), 셀러 수익 왜곡 방지 | 60건 제거됨 |
