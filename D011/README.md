# **AI 데이터 인텔리전스 부트캠프 과제: 티스토리 맞구독 분석 및 관리 자동화 파이프라인**

## **1\. 개요**

본 프로젝트는 동적 웹 페이지인 티스토리 피드에서 '나를 구독 중인 블로그'와 '내가 구독 중인 블로그' 목록을 자동으로 수집하고, 두 데이터를 비교 분석하여 효과적인 블로그 이웃 관리를 돕는 자동화 시스템을 구축하는 것을 목표로 합니다(실제로 수백개의 맞구독자가 있을 때는 관리가 어려움.). 데이터 수집, 정제, 그리고 사용자 친화적인 대시보드 생성까지의 전 과정을 구현했습니다.

## **2\. 크롤링 과정 (Data Collection)**

Playwright를 활용하여 자바스크립트 렌더링 기반의 동적 페이지를 효과적으로 제어하고 데이터를 수집했습니다.

* **세션 유지 및 안정적인 접근:**  
  * launch\_persistent\_context를 사용하여 브라우저 세션을 로컬에 저장합니다. 이를 통해 최초 로그인 이후에는 쿠키 기반의 자동 로그인이 유지되도록 구현했습니다.  
* **동적 로딩 및 렌더링 대기:**  
  * 피드 페이지 특성상 스크롤을 해야 하단 데이터가 로드되므로, window.scrollBy 스크립트를 주입하여 충분한 데이터 로딩을 유도했습니다.  
  * wait\_for\_selector 및 커스텀 대기 함수(wait\_for\_page\_items)를 통해 DOM 요소가 완전히 렌더링될 때까지 유연하게 대기하여 데이터 누락을 방지했습니다.  
* **페이징 처리 및 상태 추적 (핵심):**  
  * 단순한 버튼 클릭에 의존하지 않고, DOM 조작을 통한 정밀한 페이징을 구현했습니다.  
  * **클릭 이벤트:** page.evaluate 내에서 자바스크립트를 실행하여 현재 페이지네이션 영역의 숫자 버튼 또는 '다음' 화살표 버튼을 동적으로 탐색하고 클릭 이벤트를 발생시킵니다.  
  * **전환 감지(Transition Detection):** 버튼 클릭 후 즉시 다음 로직으로 넘어가는 것이 아니라, 활성 페이지를 나타내는 \<em\> 태그의 텍스트 변화를 폴링(Polling) 방식으로 지속적으로 감지합니다. 이 값이 목표 페이지 번호로 변경되었음이 확인된 후에만 데이터를 수집하는 강건한(Robust) 구조를 채택했습니다.

## **3\. 데이터 처리 과정 (Data Processing)**

수집된 원시 데이터(Raw HTML/DOM)에서 유의미한 URL만을 추출하고 노이즈를 제거하는 파싱 과정을 거쳤습니다.

* **타겟 데이터(URL) 추출:**  
  * 페이지 내에서 블로그 링크를 포함하는 특정 클래스(a.info\_g, a.box\_desc, a.link\_info)를 가진 엘리먼트들을 선택하고, 해당 요소들의 href 속성값을 배열로 추출했습니다.  
* **데이터 정제(Cleansing) 및 이상치 제거:**  
  * **도메인 필터링:** 추출된 주소 중 .tistory.com 문자열이 포함된 URL만 유지하고, 개인 블로그가 아닌 공식 서비스 도메인(www.tistory.com)은 필터링하여 제외했습니다.  
  * **파라미터 제거:** 사용자 환경이나 유입 경로에 따라 URL 뒤에 붙는 쿼리스트링(?), 관리자 페이지 경로(/manage), 후행 슬래시(/) 등을 문자열 분리(split)를 통해 말끔하게 제거했습니다. 이를 통해 동일 블로그가 다른 URL로 인식되는 문제를 해결했습니다.  
* **중복 데이터 병합:**  
  * 파이썬의 집합(set()) 자료구조를 사용하여 크롤링 과정에서 중복 수집된 URL들을 자동으로 제거하고 고유한(Unique) 블로그 목록만 확보했습니다.

## **4\. 데이터 분석 및 활용 (Output & Dashboard)**

정제된 데이터를 바탕으로 교집합/차집합 연산을 수행하고, 그 결과를 사용자가 쉽게 활용할 수 있도록 시각화했습니다.

* **분석 로직 (비대칭 구독자 추출):**  
  * '내가 구독 중인 목록(A)'과 '나를 구독 중인 목록(B)' 두 집합을 대상으로 차집합 연산(A \- B)을 수행했습니다. 이를 통해 "나는 구독했지만 해당 상대는 나를 구독하지 않은 블로거(짝사랑 블로거)" 목록을 정확히 산출했습니다.  
* **데이터 영속성 확보:**  
  * 분석 결과 통계와 비대칭 구독 블로거 목록을 1차적으로 텍스트 파일(tistory\_final\_report.txt) 형식으로 저장했습니다.  
* **작업 효율성을 위한 HTML 대시보드 자동화:**  
  * 파이프라인의 마지막 단계로, 저장된 텍스트 파일을 다시 읽어 들여 인터랙티브한 HTML 대시보드(tistory\_dashboard.html)를 자동 생성하는 로직을 결합했습니다.  
  * 해당 대시보드에는 타겟 블로그로 즉시 이동할 수 있는 **'방문하기' 버튼**이 제공되며, 자바스크립트 이벤트를 연동하여 클릭 시 해당 행에 취소선이 그어지게 구현했습니다. 이를 통해 사용자가 수백 개의 블로그를 방문하며 정리(구독 취소 등)하는 후속 작업을 매우 직관적이고 편리하게 수행할 수 있습니다.

## 5. 사용 방법

1. **Python 설치**: Python 3.7 이상이 설치되어 있어야 합니다.
2. **패키지 설치**: 터미널(CMD)에서 아래 명령어를 입력하여 필요한 라이브러리를 설치합니다.
   ```bash
   pip install playwright
   playwright install chromium
   ```
3. **프로그램 실행**:
   ```bash
   tistory_subscriber_manager.py
   ```
4. **브라우저 로그인**: 
   - 브라우저가 열리면 티스토리에 로그인합니다. 
   - 피드(Feed) 페이지로 이동하면 분석이 자동으로 시작됩니다.
5. **결과 확인**:
   - 터미널 창에 맞구독이 안 된 블로그 목록이 출력됩니다.
   - 상세 결과는 `tistory_follow_report.txt` 파일로도 저장됩니다.
6. make_html.py를 실행하면 대시보드 파일이 생성됩니다.

## 주요 특징

- **데이터 기반**: 단순히 화면을 긁어오는 방식이 아니라, 티스토리 내부 데이터를 직접 비교하여 매우 정확합니다.
- **안전성**: 계정 정보(비밀번호 등)를 가로채지 않으며, 오직 당신의 로컬 PC에서만 실행됩니다.
